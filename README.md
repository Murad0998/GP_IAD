# GP_IAD

# Прогнозирование продаж новых магазинов

> ML-решение для сети «Магнит» — предсказание выручки и оценка окупаемости аренды новых торговых точек с учётом дрейфа распределений.

## Описание проекта

«Магнит» — крупнейшая розничная сеть России, которая ежемесячно открывает десятки новых магазинов. Каждое решение о запуске торговой точки требует точного прогноза будущих продаж, чтобы минимизировать риски и эффективно распределять инвестиции.

**Проблема:** модель, обученная на исторических данных, со временем деградирует из-за дрейфа распределений (covariate shift и concept drift). Причины — насыщение рынка, изменение конкурентной среды, макроэкономические факторы и эволюция форматов магазинов.

**Решение:** разработана регрессионная модель, устойчивая к дрейфу, которая прогнозирует продажи новых магазинов, а также классификатор для оценки окупаемости аренды.

---

## Бизнес-задачи

### Задача A. Прогнозирование продаж (регрессия)

**Бизнес-формулировка:** предсказать выручку нового магазина до его открытия, чтобы принимать обоснованные решения о запуске точек и распределении инвестиционного бюджета.

**ML-постановка:**
- Тип: регрессия
- Целевая переменная: TARGET_1 (фактические продажи) / TARGET_2 (скорректированные)
- Метрика: MAPE
- Входные данные: геолокационные, инфраструктурные и конкурентные характеристики


### Задача B. Оценка окупаемости аренды (классификация)

**Бизнес-формулировка:** определить, окупит ли новый магазин затраты на аренду в течение первого года работы, чтобы отсеивать заведомо убыточные локации.

**ML-постановка:**
- Тип: бинарная классификация
- Целевая переменная: бинарный флаг (окупка / неокупка)
- Метрика: F1-score, precision, recall
- Бизнес-ценность: снижение операционных рисков

**Связь задач:** прогноз продаж (задача A) используется как входной признак для классификатора окупаемости (задача B).

## Данные

**Источник:** обезличенный датасет работающих торговых точек сети «Магнит».

**Разбивка:**
- `train` — обучающая выборка (признаки + TARGET_1, TARGET_2)
- `val` — валидационная выборка (признаки + TARGET_1, TARGET_2)
- `holdout` — отложенная выборка (только признаки)

**Ключевые признаки:**
- Геолокация и тип населённого пункта
- Конкурентное окружение (HuffFamilies, HUFF_RANK_COMPS_CANNIBALS)
- Инфраструктура (метро, парки, ТРЦ, больницы, школы, банки)
- Характеристики помещения (площадь, этаж, парковка)
- Арендная ставка (RENT_HEX)
